# Cover Playbook

本文件沉淀小红书封面可复用设计语法。生成方案时优先复用以下规律，再结合当前脚本做创意变体。

## 1) 高点击封面基础规律

- 使用单一核心主张，不把多个卖点并列在同一层。
- 使用大字高对比标题，首屏先读到“结果”而不是“过程”。
- 使用单一视觉主角，避免多个主体争抢注意力。
- 使用“能力隐喻”替代“产品截图堆砌”。
- 控制信息层级：主标题最大，副标题次级，补充信息最小。

## 2) 标题文案语法

- 先写结果，再写能力来源。
- 主标题优先使用“收益型”表达：提效、降本、爆发、掌控、翻倍。
- 副标题优先使用“机制型”表达：一键、自动、多视角、秒级、实战。
- 避免空泛形容词堆叠，避免无信息量感叹句。
- 严格满足主副标题各 6-10 汉字。

## 3) 视觉隐喻库

从脚本能力点映射为可视化符号：

- 自动化能力：齿轮、闪电轨迹、流水线、机械臂
- 多视角控制：环形轨道、旋转坐标、分屏镜面
- 创造力增强：能量核心、实验室、发光载体
- 效率跃迁：计时器、速度线、冲刺路径
- 成果对比：前后对照、左右分屏、层级跃迁

优先选“一个主隐喻 + 一个辅隐喻”，避免符号过载。

## 4) 配色与对比

- 背景优先深色或中性色，前景使用高亮强调色拉开对比。
- 一张封面控制在 1 个主色 + 1 个强调色 + 1 个中性色。
- 高亮色用于标题关键词、关键按钮形元素、视觉焦点。
- 保证文字和背景对比清晰，避免灰度接近导致不可读。

## 5) 版式建议（3:4）

- 画布默认 1080x1440。
- 顶部 25%-35% 区域放主副标题。
- 中下部放主视觉主体，保证轮廓完整、方向明确。
- 角标信息只放一个，避免多个贴纸干扰主视线。
- 留出安全边距，避免边缘文字被裁切。

## 6) 方案多样化策略（A/B/C）

- 方案A：结果冲击型  
  突出收益和速度，标题更直接，画面更高对比。
- 方案B：机制解释型  
  突出原理和能力过程，增加结构性辅助元素。
- 方案C：情绪代入型  
  突出人物反应或场景氛围，增强代入和好奇心。

## 7) 案例分类判定（当前案例库）

基于 `assets/examples/` 的分类目录，先判定方案走哪类风格：

- `对比效果`  
  适用：存在“前后变化、参数控制、多视角、效果证明”的脚本。  
  画面：分屏、九宫格、前后对照、箭头连接。
- `情绪表达`  
  适用：存在“震惊、惊喜、不可思议、争议观点”的脚本。  
  画面：人物夸张表情 + 大字冲击 + 单一主张。
- `插画风格`  
  适用：偏工具IP、抽象能力可视化、非真人主导的脚本。  
  画面：卡通/IP主体 + 高识别配色 + 简化背景。
- `真人形象`  
  适用：偏观点表达、职场代入、人物立场输出的脚本。  
  画面：真人近景半身像 + 强文案 + 背景辅助信息。

默认策略：三套方案至少覆盖两类；若风格偏好“冲击感”，优先含 `情绪表达` 或 `真人形象`。

## 8) 反模式清单

- 不使用过长标题或双句并列标题。
- 不使用低分辨率截图作为主视觉。
- 不将产品 UI 截图直接铺满全图。
- 不堆叠 4 种以上颜色。
- 不在同一区域放置多个同等权重文案块。

## 9) 参考库更新规则

- 仅在用户明确提出“加入案例库”或“更新风格规则”时更新本文件。
- 更新时只追加可泛化规律，不记录一次性细节。
- 每次更新优先补充“可复用语法”而非“具体画面复刻”。
